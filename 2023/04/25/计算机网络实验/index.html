计算机网络实验

Created2023-04-25|Updated2023-04-25
|Post View:

实验一：RCMS服务器

  1. 远程登录RCMS服务器（确保网线接口连接在RCMS口），浏览器输入172.16.1.10:8080，（最后一位是组号，10:第一组，20:第二组）

  1. 几种模式的变换
  1. [实验练习]依次进入D1－D4四台路由器D5－D8四台交换机，分别进入和退出路由器和交换机的各种模式。

实验二：交换机配置

  1. 在浏览器中选择二层交换机，进行连接

  1. enable 14输入密码，进入特权模式
  2. configure terminal进入全局配置模式
  3. hostname 202021091187_chenxi修改名称为学号+姓名
  4. interface vlan 1进入交换机接口配置模式

显示：Configured from outband为正常现象可以不用理会。

  1. ip address 172.16.1.xx 255.255.255.0不能设置为192.168.0.138，而且最后一位要和当前机器的ip不同，要不然后面测试连接的时候ping不通
  2. no shutdown开启交换机管理接口
  3. show ip interface验证交换机管理ip地址已经配置和开启
  5. copy running-config startup-config或write memory保存交换机配置。
  6. show configure验证交换机配置已保存。
  1. enable secret level 1 0 star设置交换机远程登录密码。
  2. line vtr 0 4
  3. 如果密码设置不成功，则跳过这步。

  1. copy running-config startup-config或write memory保存交换机配置
  2. 最后 测试验证 在cmd中输入telnet 172.16.1.xx，从PC机登录到交换机上，如下图则成功。

实验三：路由器配置

1.在浏览器中选择R2624-5-1路由器，进行连接

  1. Red-Giant>enable进入特权模式，这里路由器不需要输入star密码。
  2. configure terminal进入全局配置模式，之后会有<config>后缀。
  3. hostname 202021091187_chenxi配置路由器名称为学号+姓名。
  4. interface fastethernet0进入路由器接口配置模式。
  5. ip address 172.16.1.xx 255.255.255.0不能设置为192.168.0.138，而且最后一位要和当前机器的ip不同，要不然后面测试连接的时候ping不通。
  6. no shutdown开启路由器fastethernet0接口。
  1. show ip interface fastethernet0或show ip interface brief验证路由器接口的IP地址已经配置和开启。
  2. line vty 0 4配置路由器远程登录密码。
  3. login配置远程登录。
  4. password star设置路由器远程密码为star。
  5. end退出

  1. copy running-config startup-config或write memory保存路由器配置。
  2. show startup-config验证路由器配置已保存。
  3. 最后 测试验证 在cmd中输入telnet 172.16.1.xx，从PC机登录到路由器上，如下图则成功。

硬件跳线连接

1. 桌子底下的4个插口


  • 这里我是第二组的第4号位置，桌子底下的插口编号是207，208，RCMS，空白。
  • 首先网线连接到RCMS才能在浏览器访问连接。
  • 最后验证测试ping的时候，要把网线连接到207或208口，并且每一组的实体交换机和路由器，实体跳线的时候，网线接口要接到对应的编号。机子接的207口，交换机也要接207口，不然最后会ping不通。

2.每一组的交换机和路由器

  • 就记得第二组的实体交换器和路由器是从左往右数第四台机子了。
  • 跳线的时候，提前拿好2条网线。第一条网线接桌子底下的插口编号对应的交换机口，然后另一端接路由器。（路由器也要对应你在浏览器所选择的路由器编号）。第二条网线，要从路由器正面接到背面的fastethernet0，1，2，3口，也要对应你提前设置好的口。

以上任意接口不对应，都会导致最后ping不通。

  • fastethernet0接口，要注意，连接实体交换机的时候，可能有4个接口，要看号标号，如果设置的0口，只能插到0口，要不然最后测试验证的时候ping不通。
